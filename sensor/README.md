De code style gebaseerd op https://blog.wokwi.com/how-to-write-clean-arduino-code/  :

1.  Tussen functies en/of andere acties code een lege lijn zetten.

2.  Dan in de functies zelf zorg dat de code in de functie 1 tab heeft om het makkelijk uit te maken van functie en de code erin.

3.  Als een variabele of een functie naam en spatie moet hebben ga ik dit teken gebruiken _ in plaats van de spatie en deletters zijn allemaal kleine letters.

4.  Ik ga simpel en makkelijke variabelen namen zoveel mogelijk gebruiken.

5.  De acollodes van functies/acties gaat de begin acollade na de functie/actie en de eind accolode gaat op dezelfde afstand dat de functie heeft van de kant zijn.

6.  Gebruik geen onodige acties.

7.  Behouw notities alleen bij delen dat niet door de code kan uitgelegt worden.

8.  Voor debug code gebruik een functie zodat alleen als het nodig word gebruikt om niet heel de code te debuggen. 

9.  Gebruik gitarchive om de code dat gebarseert is op een andere code van een ander programma op te slaan zodat je later nog altijd weet waarom je dat hebt gedaan.
